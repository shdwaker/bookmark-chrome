# 全部页签功能设计

## 背景

当前插件是一个 Vue 3 + Vite + CRX 的 Chrome 书签管理扩展，主页面作为新标签页展示书签、留痕和设置入口。用户经常打开大量页面，需要一个入口查看所有当前打开的页签，并快速清理重复页签。

## 目标

新增 **“全部页签”** 功能：

- 显示浏览器当前所有已打开页签。
- 按域名分组，并用多列布局提升一屏展示量。
- 同一页面按“忽略 URL 参数和 hash 后的地址”合并展示。
- 每条页面记录显示对应页签数量。
- 支持查看重复组内的具体页面地址。
- 支持清理重复页签：保留最新打开的页签，关闭更早打开的重复页签。

## 非目标

- 不做完整标签页管理器，不支持任意拖拽、批量选择、跨窗口移动页签。
- 展开的重复明细只用于查看，不支持单个关闭或选择保留哪一个。
- 不改动现有书签、留痕、设置功能。
- 不新增 background 消息协议；当前功能由前端直接调用 Chrome Tabs API 完成。

## 入口与命名

主页面顶部导航新增按钮：**全部页签**。

点击后打开弹窗。弹窗标题也使用 **全部页签**。弹窗顶部包含：

- 当前总页签数。
- 合并后的页面记录数。
- 重复页面组数。
- 批量按钮 **清除重复**。

“清除重复”只在存在重复页面时可用；没有重复时置灰。

## 归类与重复判断

### 域名分组

一级分组按域名展示。每个域名是一张卡片，卡片右上角显示该域名下的页签总数。

无法解析标准域名的页面仍会显示：

- 插件自身页面归入插件页面分组，并永远显示在第一个位置。
- `chrome://` 等浏览器内部页面按协议或可读名称分组。
- 其他无法解析 URL 的页签归入“其他页面”。

### 页面记录合并

页面记录用规范化后的 URL 合并。规范化规则：

- 保留协议、域名和 pathname。
- 忽略 query 参数。
- 忽略 hash。

示例：

- `https://example.com/page?a=1`
- `https://example.com/page?a=2`

合并为同一条页面记录。

但以下地址仍是不同记录：

- `https://example.com/page`
- `https://example.com/other`

### 最新页签

每个页面记录中保留最新打开的页签。Chrome Tabs API 没有可靠的“打开时间”字段，因此实现需要在前端工具模块中使用可获得的顺序近似：

- 优先使用当前查询结果中更靠后的同 key 页签作为“最新”。
- 同一窗口内可参考 `tab.index`；跨窗口时按查询结果顺序处理。

清理重复时关闭该页面记录中除最新页签之外的旧页签。

## 排序

排序目标是让用户优先看到需要处理的内容，同时保留“全部页签”的浏览能力。

- 插件自身页面所在分组永远排在第一个位置。
- 其他域名卡片中，有重复页面的域名优先于无重复页面的域名。
- 域名卡片内部页面记录按重复项优先排序。
- 重复项和非重复项各自再按最新页签位置从新到旧排序。
- 数量相近时可按域名或页面标题稳定排序，避免列表跳动。

## 布局设计

弹窗使用宽屏多列布局，以适配大量打开页面的场景。

- 弹窗宽度比现有普通确认弹窗更大。
- 主体区域使用响应式网格：大屏三列，中等宽度两列，窄屏一列。
- 列表区域内部滚动，顶部统计和 **清除重复** 按钮保持可见。
- 每个域名卡片内用紧凑行展示页面记录。

页面记录行包含：

- 页面标题。
- 去掉域名后的路径或规范化地址摘要。
- 页签数量徽标。
- 当数量大于 1 时，在数量徽标右侧显示小型 **清除** 按钮。

重复记录使用浅色背景突出显示。非重复记录只显示数量 `1`，不显示清除按钮。

## 交互设计

### 打开与刷新

点击顶部导航 **全部页签** 打开弹窗。弹窗打开时读取当前所有页签。每次清理后自动重新读取并刷新列表。

### 跳转页签

点击页面记录主体时，切换到该记录对应的最新页签。

切换逻辑：

- 如果目标页签不在当前窗口，先激活目标窗口。
- 再激活目标页签。

### 查看重复明细

点击页面数量徽标展开或收起该页面记录下的具体页签明细。

明细展示：

- 页签标题。
- 完整 URL，包含 query 和 hash。
- 新旧状态标记：最新页签标记为“保留”，旧页签标记为“将清除”。

明细只用于查看，不提供单个关闭或选择保留项。

### 悬停与复制地址

鼠标悬停在页面记录或具体明细上时，用轻量悬浮层显示完整地址，并提供复制按钮。

- 复制成功显示“已复制”。
- 复制失败显示简短失败提示。
- 不使用浏览器默认 `title` 作为主要方案，因为默认提示不支持复制操作。

### 单条清除

重复记录右侧显示小型 **清除** 按钮。点击后立即关闭该记录中除最新页签之外的旧页签。

单条清除不需要二次确认，以提高操作效率。

### 批量清除

弹窗顶部按钮为 **清除重复**。点击后弹出确认，确认后对所有重复页面记录执行清理，只保留每组最新页签。

批量清理完成后自动刷新列表。

## 边界与错误处理

- 没有打开页面：显示空状态“当前没有可显示的页签”。
- 没有重复页面：仍显示全部页签；**清除重复** 按钮置灰，并显示“暂无重复页签”。
- 清理时页签已被用户手动关闭：忽略关闭失败或不存在的 tab，清理后刷新列表。
- `chrome://`、`chrome-extension://`、无 URL 页签：仍显示，不因无法解析域名而丢弃。
- 插件自身页面：显示，并永远排在第一个位置。
- 复制地址失败：显示简短失败提示。

## 技术方案

采用前端直接调用 Chrome Tabs API 的方案，不新增 background 消息协议。

### 新增 tabs 工具模块

新增工具模块负责：

- 查询所有当前页签。
- 标准化 URL。
- 提取域名或分组名称。
- 识别插件自身页面。
- 合并同一规范化 URL 的页签。
- 标记重复记录和最新页签。
- 生成 UI 所需的统计信息、域名卡片和页面记录。
- 提供关闭旧重复页签所需的 tab id 列表。

### 新增 `AllTabsModal.vue`

新增组件负责：

- 展示统计信息。
- 展示多列域名卡片。
- 展开或收起重复明细。
- 悬停显示完整 URL 并支持复制。
- 跳转到最新页签。
- 单条清除重复页签。
- 批量清除重复页签。
- 清理后自动刷新。

### 修改 `TopNav.vue`

新增 **全部页签** 按钮，并 emit `open-all-tabs` 事件。

### 修改 `App.vue`

新增 `showAllTabs` 状态，监听 `open-all-tabs` 事件并渲染 `AllTabsModal`。

## 测试计划

- 构造多个相同路径但 query 不同的页签，确认合并为一条记录。
- 构造相同域名不同路径的页签，确认分别展示。
- 确认全部页签都显示，非重复项显示为 `1` 且没有清除按钮。
- 确认重复项数量徽标可展开明细，明细显示完整 URL。
- 确认点击页面记录可切换到最新页签。
- 确认单条清除只关闭旧页签并自动刷新。
- 确认 **清除重复** 需要确认，并清理所有重复组。
- 确认没有重复时 **清除重复** 置灰。
- 确认插件自身页面显示并排在第一位。
- 确认悬停地址展示和复制地址功能。
- 运行项目构建，确保没有编译错误。
